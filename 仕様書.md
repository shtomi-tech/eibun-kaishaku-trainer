> 目的: 英文を「前から」読みながら、各かたまりの役割（S/V/O/C/M）を判定し、
> 正解するごとにそのかたまりの日本語訳が現れる＝**前から訳す力**と**構文把握**を同時に鍛えるアプリ。
> 解析（区切り・役割・訳）は **AIではなく人（先生）** が作る。

## 2. データモデル（再帰ツリー）

- **文 = 節（clause）**。節 = 前から並ぶ **chunk の列**。
- **chunk** = `{ text, role, translation, children? }`
  - `role` … `S` / `V` / `O` / `C` / `M` / `接続`（接続語＝採点対象外）
  - **leaf chunk** … `translation`（日本語）を持つ
  - **clause chunk** … `children`（内側の節）を持つ。畳んだ時の表示訳は **子の前から訳を自動連結**（先生が上書き可）
- **接続語セグメント**（`role=接続`）… 接続詞 / 関係詞 / 導く that 等は **採点対象外・訳だけ表示**。
  節を導く語は、その節chunkの先頭に含めて丸ごと1かたまりにする運用も可（先生が編集時に判断）。
- 入れ子の深さに上限は設けない（再帰モデル。実務上は2〜3階層）。

### JSON イメージ（例）

```jsonc
{
  "id": "eiken2026-1_q3_s1",
  "source": "英検2級 2026-1 大問1 Q3",
  "sentence": "Yuki only started studying Korean two years ago",
  "root": {
    "chunks": [
      { "role": "S", "text": "Yuki", "translation": "ユキは" },
      { "role": "M", "text": "only", "translation": "ほんの" },
      { "role": "V", "text": "started studying", "translation": "学び始めた" },
      { "role": "O", "text": "Korean", "translation": "韓国語を" },
      { "role": "M", "text": "two years ago", "translation": "2年前に" }
    ]
  }
}
```

clause chunk（入れ子）の例：

```jsonc
{ "role": "O", "text": "that she wanted",
  "children": { "chunks": [
    { "role": "接続", "text": "that", "translation": "（〜ということ）" },
    { "role": "S", "text": "she", "translation": "彼女が" },
    { "role": "V", "text": "wanted", "translation": "望んだ" }
  ] } }
```

## 3. プレイヤー（生徒・読み取り専用）

1. **文を選ぶ**（ソース／回ごとのリスト）。
2. **トップダウンで解く**：各 chunk で **S/V/O/C/M** を選ぶ → **即時採点**。
   - 正解 → その chunk の **訳が下に出る** → 次の chunk へ。
   - chunk が clause（節）なら、正解した瞬間に **展開して中を前から** 解析。
   - 誤答 → 「もう一度」（数回外したら正解を提示して前進）。**誤答は弱点として記録**。
   - 接続語セグメントは **採点せず、訳だけ表示**して読み流す。
3. 全レベルを正解すると **1文の前から訳が完成**。
4. **進捗（localStorage）**
   - **文ごとのクリア状況**（クリアチェッカー）
   - **役割別の誤答集計**（弱点：どの役割を間違えやすいか）
   - **生徒名（任意）**＝教室の1台共有用に切り替え。各自端末なら端末ごと。

### 表示イメージ
```
I        like     an apple
S        V        O
私は     好きだ    りんごが
```

## 4. エディタ（先生用・同一アプリ内 / 保存＝JSONエクスポート）

- **取り込み**
  - 英検2級大問1の JSON を読み、**空所 `(   )` を正解語で補完して完全文化**。
  - 2文構成の stem は **文に分割** → 解析する文を先生が選ぶ。
  - **手入力**で任意の英文も追加可。
- **区切り**：文を単語に自動分割 → **クリック/ドラッグで連結**して chunk 化 → 役割（S/V/O/C/M/接続）と訳を付与。
- **入れ子**：chunk を「節にする」→ その中で同じ操作を **再帰**。
- **保存**：`data/*.json` に **エクスポート** → コミット → **再デプロイ**で生徒に反映（フロント完結のため共有反映に再デプロイ1手間）。
- **たたき台**：英検2級大問1から **3〜5文を初期解析して同梱**（先生が確認・修正する前提）。

## 6. 実装前の改善点

### 6.1 データ契約を固定する

v1 では JSON を次の形に統一する。後で教材が増えても、`manifest.json` だけ差し替えれば一覧に出せるようにする。

```
data/
├── manifest.json
└── eiken2-2026-1.json
```

`manifest.json`

```json
{
  "datasets": [
    {
      "id": "eiken2-2026-1",
      "label": "英検2級 2026年度第1回",
      "url": "data/eiken2-2026-1.json"
    }
  ]
}
```

教材データ

```jsonc
{
  "meta": {
    "id": "eiken2-2026-1",
    "label": "英検2級 2026年度第1回",
    "source": "英検2級 2026年度第1回 大問1",
    "version": 1
  },
  "items": [
    {
      "id": "eiken2-2026-1_q3_s1",
      "source": "英検2級 2026年度第1回 大問1 Q3 / sentence 1",
      "sentence": "Yuki only started studying Korean two years ago.",
      "root": {
        "chunks": [
          { "role": "S", "text": "Yuki", "translation": "ユキは" },
          { "role": "M", "text": "only", "translation": "ほんの" },
          { "role": "V", "text": "started studying", "translation": "学び始めた" },
          { "role": "O", "text": "Korean", "translation": "韓国語を" },
          { "role": "M", "text": "two years ago", "translation": "2年前に" }
        ]
      }
    }
  ]
}
```

補足:
- `role` は `S` / `V` / `O` / `C` / `M` / `接続` のみ。
- `children` を持つ chunk は節として扱う。`translation` があれば上書き訳、なければ子 chunk の訳を前から自動連結する。
- `sentence` は表示・検索用。採点対象は必ず `root.chunks`。
- v1 では英文中の空所補完は、既存の大問1 JSON から生成した「完成文」を教材 JSON に保存しておく。

### 6.2 プレイヤーの状態遷移を明確にする

1. 画面上部で教材・生徒名を選ぶ。
2. 文リストから未クリア/弱点あり/任意の文を開く。
3. 現在 chunk だけを強調し、役割ボタン `S/V/O/C/M` を表示する。
4. `role=接続` はボタンを出さず、訳を表示して自動で次に進める。
5. 正解した chunk は `text / role / translation` を固定表示する。
6. clause chunk は正解後に展開し、子 chunk を同じルールで処理する。
7. 誤答は `mistakes[correctRole][chosenRole]` として記録する。
8. 同じ chunk を3回外したら正解を表示し、「確認して次へ」で進める。
9. root の全 chunk が完了したら文クリアにする。

### 6.3 localStorage の保存単位

共有PCで生徒名を切り替えるため、保存キーは次にする。

```
reading_trainer_v1::<datasetId>::<studentNameOrDefault>
```

保存内容:
- `clearedItemIds`: クリア済み文ID
- `itemAttempts`: 文ごとの挑戦回数・最終日時
- `mistakes`: 役割別の誤答集計
- `lastItemId`: 最後に開いた文

### 6.4 エディタ v1 の現実的な範囲

先生用エディタは同一アプリ内に置くが、v1 では「完全な制作環境」ではなく、JSONを作るための軽量補助にする。

v1 で実装する:
- 完成英文の手入力
- 単語分割
- 隣接語の結合・分割
- chunk ごとの role / translation 入力
- chunk を節化して子 chunk を編集
- JSON プレビュー
- `.json` ダウンロード
- 既存 JSON の読み込みと再編集

v1 では後回し:
- 複数文の一括インポート自動解析
- 高度なドラッグ操作
- Git commit / Netlify deploy の自動化
- 先生アカウント認証

### 6.6 v1 の合格基準

- ローカルで `py -3 -m http.server 8098` から起動できる。
- `data/manifest.json` から教材一覧を読み込める。
- 初期解析済みの3〜5文を最後まで解ける。

## 8. 確定した10の決定（`/grill-me` の記録）

1. コア体験 = **一体型・前から積み上げ**（役割判定と前から訳を1ループに統合）
2. 構文モデル = **入れ子対応**（主節／従属節をツリーで持つ）
3. 生徒の解き方 = **トップダウン展開**（節は1かたまり→正解で展開→中を前から）
4. 編集と共有 = **フロント完結＋JSON書き出し**（生徒は読み取り専用）
5. 英文ソース = **英検2級大問1を主ソース＋手入力可**（空所を正解で補完）
6. 採点 = **即時採点・正解必須で前進**（誤答は記録、数回で正解提示）
7. 編集操作 = **語をクリックで連結**してかたまり化（再帰）
8. 進捗 = **軽め進捗**（クリアチェッカー＋役割別弱点＋生徒名任意）
9. 初期データ = **ツール優先＋たたき台数文**
10. 接続語 = **非採点の「接続語」セグメント**（役割ボタンはS/V/O/C/Mの5つに保つ）
